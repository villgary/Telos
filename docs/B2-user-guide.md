## 1. 系统概述 / Product Overview

### 1.1 产品定位 / Product Positioning

Telos 是身份威胁检测与响应（ITDR）平台，聚焦账号层面的安全风险。通过自动扫描、ATT&CK 威胁映射、五层 AI 威胁分析，帮助安全运营团队发现影子账号、特权滥用、凭据泄露等安全风险。

### 1.2 核心功能 / Core Features

| 功能 | 说明 |
|------|------|
| 多源账号扫描 | SSH/Windows/数据库/云平台账号自动发现 |
| 五层威胁分析 | AI 驱动的符号层、本体层、认知层、人类学层因果层分析 |
| ATT&CK 映射 | 所有账号异常自动映射到 MITRE ATT&CK 框架 |
| NHI 管理 | 机器账号、服务账号、API Key 等非人类身份全生命周期管理 |
| 账号生命周期 | 自动识别活跃/沉睡/离职账号 |
| 合规评估 | SOC2、ISO27001、等保 2.0 一键评估 |
| 实时告警 | 多渠道（邮件/Webhook/Slack）实时告警 |

## 3. 仪表盘 / Dashboard

路径：**首页**

仪表盘分为**运营视图**和**管理视图**，由右上角视图切换控制。

### 3.1 运营视图 / Operator Dashboard

适合日常运营人员，聚焦当前安全状态：

**关键指标卡片：**
- **风险评分**：全局风险评分（0-100），颜色标识（绿<20 <黄<45 <橙<70 <红）
- **总资产数**：已纳入扫描的资产总数
- **总账号数**：所有资产上发现的账号总数
- **特权账号数**：拥有管理权限的账号数量
- **合规覆盖率**：最近 30 天有扫描记录的资产比例

**风险趋势图**：最近 7/30 天告警趋势

**账号统计：**按账号类型（管理员/普通/服务/System）分类饼图

**最近告警：**最新 10 条告警列表，点击可跳转详情

### 3.2 管理视图 / Admin Dashboard

适合安全管理员，聚焦全局态势：

- **横向移动风险热点**：按资产分组显示风险最高的资产
- **账号风险分布**：按风险级别（严重/高危/中危/低危）分布
- **ATT&CK 战术覆盖**：各战术阶段的信号数量
- **扫描健康状态**：最近扫描作业状态一览
- **AI 摘要**：AI 自动生成的威胁态势报告

## 4. 资产管理 / Asset Management

### 4.1 资产列表 / Asset List

路径：**资产管理 → 资产列表**

**列表字段：**

| 字段 | 说明 |
|------|------|
| 资产编码 | 唯一标识，如 `WEB01` |
| IP 地址 | 服务器 IP |
| 操作系统 | Linux / Windows / Database 等 |
| 账号数 | 最近一次扫描发现的账号数量 |
| 特权账号 | 特权账号数量（颜色编码）|
| 最后扫描 | 最近扫描时间 |
| 状态 | 在线/离线/扫描中 |

**操作按钮：**
- **扫描**：对选中资产发起扫描
- **编辑**：修改资产信息
- **删除**：移除资产（同时删除扫描记录）
- **导出**：导出 CSV

### 4.2 添加资产 / Add Asset

点击**添加资产**按钮，填写表单：

| 字段 | 必填 | 说明 |
|------|------|------|
| 资产名称 | ✅ | 显示名称，如 `web-server-01` |
| 资产编码 | ✅ | 唯一编码，如 `WEB01` |
| IP 地址 | ✅ | IPv4 地址 |
| 操作系统 | ✅ | Linux / Windows / Database / Cloud |
| 类别 | ✅ | 选择已有类别或新建 |
| 资产分组 | | 可选，用于批量管理 |
| 上级资产 | | 可选，构建资产拓扑关系 |
| 凭据 | ✅ | 选择连接该资产所需的凭据 |

### 4.3 资产类别 / Asset Categories

路径：**资产管理 → 资产类别**

层级式资产分类系统：
- **Server / Linux**：Linux 服务器，触发 SSH 扫描
- **Server / Windows**：Windows 服务器，触发 WinRM 扫描
- **Database / MySQL**、**Database / PostgreSQL**、**Database / Redis** 等
- **Cloud / 云服务器**：云平台 ECS 等

每个类别可单独设置**沉睡阈值**（天数）。

### 4.4 资产拓扑 / Asset Topology

路径：**资产管理 → 资产拓扑**

可视化展示资产层级关系（上下级关系）：
- 父级资产（机房、机架）→ 子级资产（具体服务器）
- 鼠标悬停显示账号信息
- 支持按风险高低着色

## 5. 扫描作业 / Scan Jobs

### 5.1 扫描任务 / Scan Tasks

路径：**扫描作业 → 扫描任务**

**发起扫描：**
1. 进入扫描任务页面
2. 点击**新建扫描**
3. 选择扫描范围：
   - **按资产选择**：勾选具体资产
   - **按分组选择**：按资产分组批量选择
   - **按类别选择**：如"所有 Linux 服务器"
4. 选择**全量扫描**（发现所有账号）或**增量扫描**（仅上次扫描后新增账号）
5. 点击**开始扫描**

**扫描进度：**
- 实时显示进度条
- 显示已发现账号数量
- 扫描完成后显示摘要

**扫描结果字段：**

| 字段 | 说明 |
|------|------|
| 用户名 | 系统账号名 |
| UID/SID | 系统标识符 |
| 主组 | 主要所属组 |
| Shell | 登录 Shell（/bin/bash 等）|
| 主目录 | Home 目录路径 |
| 账号状态 | enabled / disabled / locked |
| 是否管理员 | 是否属于特权组（sudo/admin）|
| 最后登录 | 最近登录时间 |
| 特权配置 | sudoers 内容摘要 |

### 5.2 定时扫描 / Scheduled Scans

路径：**扫描作业 → 定时扫描**

配置周期性自动扫描：

| 预设 | Cron 表达式 |
|------|-----------|
| 每小时 | `0 * * * *` |
| 每天凌晨 3 点 | `0 3 * * *` |
| 每周一凌晨 3 点 | `0 3 * * 1` |
| 每月 1 日凌晨 3 点 | `0 3 1 * *` |
| 自定义 | 自定义 Cron 表达式 |

### 5.3 Diff 对比 / Diff View

路径：**扫描作业 → Diff 对比**

对比任意两次扫描结果，检测变更：

| 变更类型 | 风险级别 | 说明 |
|---------|---------|------|
| 新增账号 | 🟡 中危 | 之前不存在的新账号 |
| 删除账号 | 🟢 低危 | 已消失的账号 |
| 权限提升 | 🔴 高危 | 普通账号变为管理员 |
| 权限降级 | 🔵 低危 | 管理员被降为普通账号 |
| 配置变更 | 🟡 中危 | Shell/状态等配置变化 |

## 6. 身份威胁分析 / Identity Threat Analysis

路径：**身份威胁分析**

### 6.1 五层分析 / Five-Layer Analysis

| 层级 | 检测内容 | 典型信号 |
|------|---------|---------|
| **符号层** | 账号命名异常 | 字符伪装（l→1, o→0）、匿名命名（_admin）、符号模仿 |
| **本体层** | 身份混淆与孤立 | 影子账号、孤儿账号、同名不同人 |
| **认知层** | 认知盲点 | 服务账号用交互 Shell、特权账号无登录 |
| **人类学层** | 组织关系异常 | 高风险信任链、权限簇人机混用 |
| **因果层** | 深层因果关系 | 权限提升路径、沉睡特权链、因果中枢 |

### 6.2 执行分析 / Run Analysis

1. 选择分析范围：
   - **全局分析**：所有资产的所有账号
   - **单资产分析**：仅某一台服务器的账号
   - **单身份分析**：某一关联身份下的所有账号
2. 选择分析引擎：**Go 引擎**（推荐，高性能）或 **Python 引擎**
3. 点击**开始分析**
4. 等待分析完成（实时显示进度和分析耗时）

### 6.3 攻击路径模拟 / What-If Attack Simulation

在分析结果页面选择**攻击模拟**：

1. **选择 Patient Zero（0号病人）**：选择一个可能被攻击者拿下的账号
2. **设置跳数上限**：BFS 搜索深度（建议 3-5 跳）
3. **点击模拟**：系统自动分析攻击扩散范围

**模拟结果：**
- 可达的管理员账号列表
- 可达的高风险资产列表
- 攻击路径详情（每跳的账号和资产）
- 关联的 ATT&CK 技术

### 6.4 导出 ATT&CK Layer 文件 / Export ATT&CK Layer

在分析结果页面点击**导出 Layer**，生成 JSON 文件，可导入 [MITRE ATT&CK Navigator](https://mitre-attack.github.io/attack-navigator/) 可视化。

## 7. ATT&CK 覆盖率 / ATT&CK Coverage

路径：**ATT&CK 覆盖率**

### 7.1 功能说明 / Overview

ATT&CK 覆盖率页面将分析结果映射到 MITRE ATT&CK 框架，以可视化方式展示当前环境中的账号威胁态势。

### 7.2 两个视图 / Two Views

**视图一：风险摘要 / Risk Summary**
- 按严重性（严重/高危/中危/低危）分组展示所有信号
- 每条信号显示：信号类型、影响账号、ATT&CK ID、处置建议
- 适合快速了解最高风险项

**视图二：ATT&CK 技术详情 / Technique Details**
- 按 MITRE ATT&CK 战术和技术分组
- 显示每个技术的置信度、信号数量、证据摘要
- 适合深度分析和报告

### 7.3 ATT&CK 战术说明 / ATT&CK Tactics

| 战术 ID | 战术名称 | 说明 |
|---------|---------|------|
| TA0001 | 初始访问 | 攻击者如何获取初始立足点 |
| TA0002 | 执行 | 执行恶意代码的能力 |
| TA0003 | 持久化 | 在系统上保持持久性的方法 |
| TA0004 | 权限提升 | 获取更高权限的方法 |
| TA0005 | 防御规避 | 规避安全检测的方法 |
| TA0006 | 凭据访问 | 访问凭据的技术 |
| TA0007 | 发现 | 了解环境和账号结构 |
| TA0008 | 横向移动 | 在网络中移动的能力 |
| TA0009 | 收集 | 从系统中收集数据 |
| TA0010 | 命令与控制 | 远程控制能力 |
| TA0011 | 数据窃取 | 从系统外传数据的能力 |
| TA0040 | 影响 | 对系统和数据造成影响 |

## 8. 告警中心 / Alert Center

路径：**安全运营 → 告警中心**

### 8.1 告警类型 / Alert Types

| 类型 | 级别 | 触发条件 |
|------|------|---------|
| 新增管理权限 | 🟠 高危 | 普通账号变为管理员 |
| 新增 NOPASSWD sudo | 🔴 严重 | 账号新增无密码 sudo 权限 |
| 敏感凭据泄露 | 🔴 严重 | 发现危险凭据文件（shadow、known_hosts 等）|
| 休眠账号重新激活 | 🟡 中危 | 之前标记为休眠/离职的账号重新活跃 |
| 孤儿特权账号 | 🟠 高危 | 特权账号无关联人员身份（影子账号）|

### 8.2 告警工作流 / Alert Workflow

```
新告警 (New) → 已确认 (Acknowledged) → 已响应 (Responded)
                             ↘ 已驳回 (Dismissed)
```

**操作说明：**

| 操作 | 效果 |
|------|------|
| **确认** | 标记为已知，停止通知 |
| **响应** | 已处理完毕，生成审计记录 |
| **驳回** | 误报，标记为无效 |

### 8.3 实时告警流 / Real-Time Alert Stream

告警中心支持 SSE 实时推送，新告警到达时：
- 右上角铃铛徽章数字更新
- 下拉列表显示最新告警预览
- 页面内实时刷新列表

## 9. 账号生命周期 / Account Lifecycle

路径：**安全运营 → 账号生命周期**

### 9.1 状态类型 / Status Types

| 状态 | 定义 | 颜色 |
|------|------|------|
| **活跃 / Active** | 30 天内有登录行为 | 🟢 绿色 |
| **沉睡 / Dormant** | 超过活跃天数但未超离职阈值 | 🟡 黄色 |
| **离职 / Departed** | 超过离职天数阈值 | 🔴 红色 |
| **未知 / Unknown** | 从未扫描到登录记录 | ⚪ 灰色 |

### 9.2 配置阈值 / Configure Thresholds

**全局阈值**（适用于所有资产）：
- 活跃天数阈值（默认 90 天）
- 离职天数阈值（默认 180 天）

**按类别阈值**：
- 为不同资产类别设置不同阈值（如数据库账号可设置更长）

### 9.3 状态历史 / Status History

每个账号的状态变更历史，记录：
- 状态切换时间
- 切换原因（自动判断/人工调整）
- 关联扫描记录

## 10. NHI 非人类身份管理 / NHI Management

路径：**NHI**

### 10.1 NHI 类型 / NHI Types

| 类型 | 说明 |
|------|------|
| **Service Account** | 服务账号（进程/服务使用）|
| **System Account** | 系统账号（OS 内置）|
| **Cloud Identity** | 云 IAM 身份/角色 |
| **Workload** | 工作负载账号 |
| **CI/CD** | CI/CD Pipeline 凭证 |
| **Application** | 应用程序账号 |
| **API Key** | API 密钥凭证 |
| **AI Agent** | AI Agent 凭证 |
| **Unknown** | 未能分类的账号 |

### 10.2 NHI 仪表板 / NHI Dashboard

- **Top 10 高风险 NHI**：按风险评分排序
- **NHI 类型分布**：饼图展示各类型占比
- **风险级别分布**：严重/高危/中危/低危占比

### 10.3 分配归属人 / Assign Owner

为 NHI 分配归属人（邮箱），系统自动关联到人员身份融合库：
1. 点击 NHI 右侧**分配归属人**
2. 输入归属人邮箱
3. 系统自动查找关联的人员身份
4. 关联后在身份融合页面可查看该人员所有关联账号

### 10.4 轮换提醒 / Rotation Reminders

跟踪 NHI 凭证轮换周期：
- 设置每类 NHI 的轮换天数阈值
- 系统自动计算距下次轮换剩余天数
- 即将到期（≤ 30 天）触发告警

## 11. 合规中心 / Compliance Center

路径：**安全运营 → 合规中心**

### 11.1 支持的合规框架 / Supported Frameworks

| 框架 | 说明 |
|------|------|
| **SOC 2** | Trust Services Criteria |
| **ISO 27001** | 信息安全管理体系 |
| **等保 2.0** | 中国网络安全等级保护 |

### 11.2 合规检查项 / Compliance Checks

**SOC 2：**
- CC6.1：共享特权账号检测
- CC6.3：未使用特权账号清理（90 天+无登录）
- CC6.6：定期访问审查（30 天内扫描覆盖率）

**ISO 27001：**
- A.9.2.3：禁止无密码 sudo
- A.9.4.3：特权账号比例（建议 < 50%）
- A.9.2.5：认证失败处理

**等保 2.0：**
- DBAP_TS：离线资产特权账号检测
- DBAP_OA：静默管理员账号审查
- CC6.3：长期不活跃特权账号

### 11.3 执行合规评估 / Run Compliance Assessment

1. 进入合规中心
2. 选择框架（SOC2 / ISO27001 / 等保2.0）
3. 选择评估范围（全部资产或指定分组）
4. 点击**执行评估**
5. 等待完成后查看结果（通过率、总评分）

## 12. 知识库 / Knowledge Base

路径：**安全运营 → 知识库**

### 12.1 功能说明 / Overview

基于 RAG（检索增强生成）的安全知识库，支持自然语言提问：

- **MITRE ATT&CK 战术与技术**：完整的 ATT&CK Framework 知识
- **CVE 漏洞库**：漏洞编号、严重性、受影响版本
- **安全最佳实践**：检测提示、缓解方法

### 12.2 智能问答 / Ask Questions

在知识库页面输入自然语言问题，例如：
- "SSH 密钥复用如何检测？"
- "T1078.004 是什么攻击技术？"
- "如何配置 NOPASSWD sudo 告警？"

## 13. 常见问题 / FAQ

**Q: 扫描时提示"连接超时"怎么办？**
A: 检查以下几点：① 目标服务器防火墙是否放行 SSH 22 端口；② 凭据（用户名/密码/SSH 密钥）是否正确；③ Control Plane 与目标服务器网络是否互通（ping/telnet 测试）。

**Q: 告警没有触发，是什么问题？**
A: 确认：① 告警类型在实时监控范围内；② 通知渠道已配置（Webhook/邮件）；③ 告警未被静默规则屏蔽。

**Q: 五层分析结果为空是什么原因？**
A: 可能原因：① 尚未执行任何扫描（需要先扫描）；② 分析引擎异常（查看日志）。

**Q: ATT&CK 覆盖率显示为 0 如何处理？**
A: 需要先在**身份威胁分析**页面执行一次分析，分析完成后 ATT&CK 覆盖率页面才会显示数据。

**Q: 如何修改用户密码？**
A: 右上角用户菜单 → **个人设置** → **修改密码**，输入旧密码和新密码后提交。
